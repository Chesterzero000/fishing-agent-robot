# Skill: 生成双平台推送内容

## 描述

根据给定的钓鱼知识点，生成适用于公众号和小红书的双平台文案内容。

## 使用方法

```bash
/generate-post
```

然后按提示输入：
- 知识点主题（如：调漂技巧）
- 目标受众等级（入门/进阶/高级）

## 工作流程

1. **获取知识点内容**
   - 从 MySQL 数据库查询相关知识点
   - 或接受用户直接输入的知识内容

2. **生成公众号文案**
   - 标题格式：`【小沄学钓鱼】XXX的正确姿势`
   - 开篇：小沄的困惑引入
   - 正文：详细知识讲解（800-1500字）
   - 结尾：要点总结 + 互动话题

3. **生成小红书文案**
   - 标题格式：`新手必看！XXX原来这么简单 🎣`
   - 正文：简短文案（200-400字）
   - 风格：口语化 + emoji + 互动引导

4. **生成6格漫画脚本**
   - 第1格：小沄遇到问题/疑惑
   - 第2格：尝试解决/学习中
   - 第3格：发现关键知识点
   - 第4格：实践应用
   - 第5格：遇到小波折或惊喜
   - 第6格：成功/总结/彩蛋

```markdown
## 知识点：[主题]

---

### 📱 公众号版

**标题**：【小沄学钓鱼】XXX的正确姿势

**内容**：
[详细内容...]

---

### 📕 小红书版

**标题**：新手必看！XXX原来这么简单 🎣

**内容**：
[简短内容...]

| 格子 | 画面描述 | 对话/旁白 |
|------|----------|-----------|
| 第1格 | 小沄[表情]在[场景] | "..." |

```

## 提示词模板

### 公众号生成提示词

```
你是一位钓鱼知识科普专家，为南科大钓鱼协会的公众号撰写内容。

IP形象：小沄 - 圆滚滚的荔枝形状吉祥物，呆萌爱学习。

知识点：{knowledge_topic}

请按照以下结构生成内容（800-1500字）：

1. **开篇引入**（100字）
   - 小沄在钓鱼时遇到的困惑
   - 自然引出今日主题

2. **知识详解**（600-1200字）
   - 系统讲解知识点
   - 分步骤说明
   - 配合实际案例

3. **要点总结**（100字）
   - 3-5条核心要点

4. **互动话题**（50字）
   - 引导读者评论分享

风格：专业、系统、有深度
```

### 小红书生成提示词

```
你是一位钓鱼知识博主，为南科大钓鱼协会的小红书账号撰写内容。

知识点：{knowledge_topic}

请按照以下结构生成内容（200-400字）：

1. **开场**（50字）
   - 亲切问候 + 引入话题

2. **干货讲解**（100-200字）
   - 用大白话讲知识点
   - 多用比喻和例子

3. **要点总结**（50-100字）
   - 带emoji的要点列表

4. **互动引导**（50字）
   - 引导收藏/点赞/评论

风格：轻松、口语化、多用emoji（🎣🐟💪）

示例标题：
- 新手必看！调漂原来这么简单 🎣
- 钓鱼人必学！3步搞定开饵 💪
- 路亚入门！选对拟饵鱼就来了 🐟
```
